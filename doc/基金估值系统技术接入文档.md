## 1. 系统概述

### 1.1 项目简介

**real-time-fund** 是一个基于 Next.js 开发的**纯前端基金估值与重仓股实时追踪工具**，采用玻璃拟态设计（Glassmorphism），完美适配 PC 与移动端。

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| 实时估值 | 通过 JSONP 直连天天基金，获取实时估算净值 |
| 重仓追踪 | 自动解析东方财富 HTML，展示前10大重仓股 |
| 纯前端运行 | 无需后端服务器，支持 GitHub Pages 部署 |
| 本地持久化 | localStorage 存储所有用户数据 |
| 云端同步 | 可选 Supabase 多设备数据同步 |
| 响应式设计 | PC/移动端自适应布局 |

### 1.3 在线预览

- 主站：https://hzm0321.github.io/real-time-fund/
- 加速：https://fund.cc.cd/

### 4.2 关键架构决策

#### 为什么是纯前端？

| 决策点 | 方案 | 选择原因 |
|--------|------|----------|
| 后端服务 | ❌ 无需 | 减少运维成本 |
| 数据获取 | ✅ JSONP | 绕过CORS限制 |
| 数据存储 | ✅ localStorage | 无需数据库 |
| 部署方式 | ✅ SSG静态站 | 支持GitHub Pages免费托管 |

#### JSONP 原理说明

```javascript
// 传统 AJAX 会遇到 CORS 错误
fetch('https://fundgz.1234567.com.cn/js/000001.js') 
// ❌ Error: CORS policy blocked

// JSONP 通过 <script> 标签绕过限制
const script = document.createElement('script');
script.src = 'https://fundgz.1234567.com.cn/js/000001.js?callback=jsonpgz';
// 服务器返回: jsonpgz({ fundcode: "000001", name: "xxx", gsz: "1.2345", ... })
document.body.appendChild(script); // ✅ 成功执行回调
```

### 6.3 缓存策略

**文件**: `app/lib/cacheRequest.js`

```javascript
// 三级缓存策略
const _cacheResultMap = new Map();      // 已完成的请求结果
const _cachePendingPromiseMap = new Map(); // 进行中的请求

export async function cachedRequest(request, cacheKey, options) {
  // 1. 有缓存直接返回
  if (_cacheResultMap.has(cacheKey)) {
    return _cacheResultMap.get(cacheKey);
  }
  
  // 2. 相同请求正在进行中，复用 Promise
  if (_cachePendingPromiseMap.has(cacheKey)) {
    return await _cachePendingPromiseMap.get(cacheKey);
  }
  
  // 3. 发起新请求
  const promise = request();
  _cachePendingPromiseMap.set(cacheKey, promise);
  const result = await promise;
  
  // 4. 写入缓存，设置过期时间
  _cacheResultMap.set(cacheKey, result);
  setTimeout(() => _cacheResultMap.delete(cacheKey), cacheTime);
  
  return result;
}
```

| 数据类型 | 缓存时间 | 说明 |
|----------|----------|------|
| 基金估值 | 无缓存（每次刷新） | 实时性要求高 |
| 历史净值 | 10分钟 | 变化频率低 |
| 重仓股 | 1小时 | 季度披露 |
| 板块行情 | 1分钟 | 实时性要求 |
| 关联板块 | 1天 | 低频变化 |

### 7.2 解析流程

```javascript
// fund.js 中的解析逻辑
const parseNetValuesFromLsjzContent = (content) => {
  // 1. HTML内容提取
  const rowMatches = content.match(/<tr[\s\S]*?<\/tr>/gi);
  
  // 2. 逐行解析
  for (const row of rowMatches) {
    const cells = row.match(/<td[^>]*>(.*?)<\/td>/gi);
    
    // 3. 提取字段
    date: cells[0],    // 日期 "2024-01-15"
    nav: cells[1],     // 净值 "3.4567"
    growth: cells[?],  // 日涨幅 "+0.94%"
  }
  
  // 4. 返回升序排列的结果
  return results.reverse();
};
```

```javascript
  const share = holding.share;           // 持有份额
  const yesterdayNavDelta = fund.yesterdayNavDelta;  // 昨日净值差
  return share * yesterdayNavDelta;     // 当日收益 ≈ 份额 × 净值差
};
```

### 12.8 风险与应对

| 风险 | 影响 | 应对措施 |

| **接口稳定性** | 公开接口可能变更 | 封装抽象层、多数据源备选 |

### C. 常见问题

**Q1: 为什么选择 JSONP 而不是 Fetch/Ajax？**
A: 因为公开的金融接口大多没有开启 CORS，JSONP 通过 `<script>` 标签可以绕过跨域限制。

**Q2: 数据存储在浏览器安全吗？**
A: localStorage 是明文存储，不建议存放敏感信息。本项目仅存储基金代码和公开数据。

**Q3: 如何处理大量基金的刷新？**
A: 使用 `Promise.all` 并发请求，配合 `cachedRequest` 避免重复请求。建议控制单次刷新基金数量在50只以内。

**Q4: 支持哪些浏览器？**
A: 支持所有现代浏览器（Chrome、Firefox、Safari、Edge最新两个版本）。IE不支持。
